  2. 后台管理平台新增一个『话题管理』页，可以新增话题、将话题设为推荐、删除话题等

糖糖圈消息收纳机制

发表于 2016-11-14

糖糖圈目前的消息通知分散在两处：

  • 首页右上角的系统消息和医院通知管理收纳
  • 糖圈左上角的社交通知收纳，包括评论、赞、@求助

目前的问题：

  1. 社交通知混杂在一起，会对用户的理解造成困扰
  2. 通知的提醒有时不是很及时
  3. 用『@』为按钮名称不是很明确，可以直接改为『消息』
  4. 两处通知怎么做明确的类型区隔
  5. 还有些有用通知信息现在还没加入到通知机制里，如『商城发货信息』等

解决方案：

参照一些通知分类式管理，这样虽然增加了一级操作，但是整体的观感更明确，用户的理解成本更低，也易于后期增加更多类型通知。
目前的消息类型如下：

  • 回复：
  • @求助
  • 赞：

糖糖圈的发展想法

发表于 2016-10-27

糖糖圈APP在目前阶段从格局和功能需求两方面看应该都要有突破性的进展才能更进一步，进入医疗领域第一梯队的层次。

从功能需求上来看，v2.5版本的更新补足了之前APP所缺失的部分，让整个软件的可用性提升了一个台阶，但是新增加的功能还存在实用性上的不断优化，而且对于核心的血糖记录及血糖趋势分析部分未做探索改进，这在用户体验上还是存在很大缺失和隔阂的。

::v2.5的更新内容::

  1. 用户成长体系
  2. 商城系统
  3. 多媒体内容
  4. 吉祥物多样化、智能化的探索
  5. 软件界面的美化和易用化

从行业格局来看，低头优化认真优化产品需要做，但是也要抬头看行业的发展和趋势，单纯的靠软件的新颖功能是否能够突破糖糖圈目前的瓶颈，我们的产品影响力和团队运营能力对比其他同类竞品是否有竞争力

::战略上需要发力的点::

  1. 糖糖圈品牌影响力的塑造
  2. 有效的拉新能力
  3. 爆点内容的制作能力

目前我们的产品体系应该是包含我们的APP、合作的硬件产品、及我们能覆盖的用户群（包括微信群及其他渠道）、合作机构；
最终的沉淀及整合平台还是要放到APP上来的，那么这些资源怎么做有机的结合和运作就是关键了，能否在有效的医学模型下建立社会化的实践方法并产出作用是我们应该做的探索。

软件体验和功能上的改进方向，需要有一个明确的迭代周期时间表和下一个阶段逐步需要实现的功能台阶，因为很多功能都可以整合进来，但是进度和加入方式需要结合我们用户的特定人群。

::功能迭代阶梯::

  1. 用户之间的IM功能 ⭐⭐⭐
  2. 血糖记录体验优化 ⭐⭐⭐
  3. 媒体内容的大整合，包括文章、食物、多媒体 ⭐⭐⭐
  4. 成熟商城及支付系统的搭建 ⭐⭐
  5. 语音直播功能，移植糖糖圈等优秀节目 ⭐⭐

APP的迭代效率需要提升，大功能的新增两月一个，小版本及Bug优化2-3周一次更新，医院端的同步要一致。
同时，运营部门要配合好新版本更新的推进，保证活跃用户都已经体验到完整的最新功能并能给出有效反馈，进而给出后续版本改进的意见。

title:Xcode自动布局的学习点记录

1、swift通过cocoapod导入第三方库的方式与oc是相同的

platform :ios, '8.0'
use_frameworks!

target 'MyApp' do
    pod 'AFNetworking', '~> 2.6'
    pod 'ORStackView', '~> 3.0'
    pod 'SwiftyJSON', '~> 2.3'
end

2、自动布局的原则是，固定一个锚点，这个固定组件可以是label或者一个view，但是要与母View进行位置绑定；然后根据锚点再依次布局其他组件

3、一个组件布局的基本属性：宽／高、和上一个组件的位置关系、有center属性，可以用来在某些需要居中的情况

4、segue也有属性，比如presentation可以决定下个页面是怎样形式来出现

5、一般为每一个View都建立一个Class

6、先将View绑定到Class，然后再将组件通过IBOutlet绑定到Class中

7、storyboard里的模型尺寸是1/2的真机像素尺寸；所以sketch输出0.5x拖入storyboard显示效果和sketch里1x是一样的

2、id为任意类型，方法默认返回id类型
3、使用ARC创建和初始化对象使用代码:[[myObject alloc] init]
4、方法的声明:+ (NSNumber *)numberWithBool: (BOOL)value;
5、方法的调用:[NSNumber numberwithBool]

在接口中声明类

1、调用超类的方法：[super xx]; xx为方法名
2、声明类的关键词为@interface, 基本写法：
@interface XXX: NSObject
@end
3、声明未声明的类使用@class XXX
4、引用未声明的协议用@protocol XXX
5、类实现的关键词为@implementation

1、声明属性的关键词为@property,基本写法：@property id xx,id为属性类型，比如整型、浮点型等，xx为属性名
2、属性实现中的关键词为:@synthesize
